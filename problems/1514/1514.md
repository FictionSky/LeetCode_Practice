# 1514. Path with Maximum Probability

## 题目链接

- https://leetcode.com/problems/path-with-maximum-probability/

## 题目要求

给定 `n`, `edges`, `succProb`, `start_node`, `end_node`，完成 `Path with Maximum Probability` 对应的 `maxProbability` 逻辑，并返回题目要求的结果。

## 示例

### 示例 1

- 输入：`n = 3, edges = [[0, 1], [1, 2], [0, 2]], succProb = [0.5, 0.5, 0.2], start_node = 0, end_node = 2`
- 输出：`0.25`
- 解释：本地样例用于验证你的实现是否符合题目核心规则。

## 约束条件

- `n` 必须按原题含义处理。
- `edges` 必须按原题含义处理。
- `succProb` 必须按原题含义处理。
- `start_node` 必须按原题含义处理。
- `end_node` 必须按原题含义处理。
- 注意空输入、重复值、边界下标和结果顺序等边界情况。

## 解题思路

- 先定义点、边和路径代价。
- 根据边权特征选择 Dijkstra、Bellman-Ford 或二分 + BFS。

## 推荐题型

- 主模式：`带权图 / 最短路`

## 提示

- 在选择 BFS、Dijkstra 或 DP 前，先定义清楚“状态”是什么。
- 注意某条路径代价何时已经确定，何时还可能继续变优。

## 复杂度目标

- 如果使用优先队列，常见目标是 O((V + E) log V)。
