# 406. Queue Reconstruction by Height

## 题目链接

- https://leetcode.com/problems/queue-reconstruction-by-height/

## 题目要求

给定 `people`，完成 `Queue Reconstruction by Height` 对应的 `reconstructQueue` 逻辑，并返回题目要求的结果。

## 示例

### 示例 1

- 输入：`people = [[7, 0], [4, 4], [7, 1], [5, 0], [6, 1], [5, 2]]`
- 输出：`[[5, 0], [7, 0], [5, 2], [6, 1], [4, 4], [7, 1]]`
- 解释：本地样例用于验证你的实现是否符合题目核心规则。

## 约束条件

- `people` 必须按原题含义处理。
- 注意空输入、重复值、边界下标和结果顺序等边界情况。

## 解题思路

- 先写出局部最优选择。
- 再确认这个选择不会破坏后续最优解。

## 推荐题型

- 主模式：`贪心`

## 提示

- 写代码前先明确贪心选择到底是什么。
- 解释清楚为什么当前局部最优不会破坏全局最优。

## 复杂度目标

- 很多贪心题的常见目标是 O(n) 或 O(n log n)。
