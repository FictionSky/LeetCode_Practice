# 278. First Bad Version

## 题目链接

- https://leetcode.com/problems/first-bad-version/

## 题目要求

版本从 1 到 `n` 排列，某个版本开始变坏，之后所有版本都坏；通过 `isBadVersion` 找出第一个坏版本。

## 示例

### 示例 1

- 输入：`n = 5, bad = 4`
- 输出：`4`
- 解释：版本 4 和 5 坏，版本 1 到 3 好。

## 约束条件

- 只能通过 API 判断某个版本是否坏。
- 应尽量减少 API 调用次数。

## 推荐题型

- 主模式：`二分查找`

## 提示

## 复杂度目标

- 优先尝试 O(log n) 时间复杂度。
