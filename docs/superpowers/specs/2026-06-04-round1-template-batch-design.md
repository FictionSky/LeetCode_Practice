# Round 1 模板批量生成设计

## 目标

把 Round 1 的 75 道题统一生成成可复用的练习模板，并保证目录结构、命名规则、文件内容规则全部一致。

## 范围

本次只覆盖 `docs/leetcode-3-round-plan.md` 中的 Round 1 题目。

不处理：

- Round 2
- Round 3
- 完整题解代码
- 自动抓取完整题面

## 目录结构

所有题目统一放在 `problems/` 下，例如：

```text
problems/
└── 020/
    ├── 020.md
    └── 020_ValidParentheses.cpp
```

命名规则固定为：

- 文件夹：三位补零题号，例如 `020`
- Markdown：`<题号>.md`
- C++：`<题号>_<英文题名>.cpp`

## 内容规则

### Markdown 文件

每道题的 `md` 文件都使用中文，并包含：

- 题号和标题
- 题目要求
- 示例
- 约束
- 提示
- 复杂度目标

### C++ 文件

每道题的 `cpp` 文件必须：

- 保留 `Solution` 类
- 保留目标函数签名
- 不提供完整答案
- 提供清晰的 `TODO` 中文注释
- 提供帮助用户快速定位答题区域的中文说明
- 保留本地 `main`
- 运行输出使用中文

## 生成策略

使用“题目元数据 + 统一模板”的方式批量生成，避免手工维护几十组目录和文件。

生成时：

- 保留已经确认过的 `001`
- 对其余题目统一生成
- 后续如规则变化，可按同一模板重新覆盖生成

## 验证

完成后至少检查：

- 代表题目录存在
- 每题都有 `md + cpp`
- `CMake` 仍能扫描 `problems/*.cpp`
- 抽查模板程序可以编译运行
