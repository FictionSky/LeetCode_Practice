# LeetCode 练习模板设计

## 目标

给每道 LeetCode 题目准备统一、轻量、可复用的练习模板，帮助用户自己写答案，而不是直接拿到现成题解。

## 范围

先从 `001` 的模板规则开始确定，并把这套规则作为后续所有题目的默认结构。

## 采用方案

每道题统一放在 `problems/` 下，使用下面的结构：

- `problems/<题号>/<题号>.md`
- `problems/<题号>/<题号>_<英文题名>.cpp`

其中：

- `md` 文件使用中文
- `cpp` 文件中的注释使用中文
- `main` 的本地运行提示使用中文
- `cpp` 文件保持“未作答练习模板”状态

## 为什么这样设计

- 用户可以自己完成算法实现，真正训练解题能力
- 模板仍然可以本地编译运行，方便调试
- 中文说明更适合当前仓库的使用场景
- 所有题目的结构统一，后续复习和扩展都更轻松

## 文件规则

### Markdown 文件

应该包含：

- 题号和标题
- 中文题目要求
- 示例占位
- 约束占位
- 解题提示
- 复杂度目标

### C++ 模板文件

应该包含：

- 必要头文件
- `Solution` 或设计题要求的类
- 清晰的 `TODO` 区域
- 中文注释，明确告诉用户在哪里补代码
- 本地 `main`
- 中文输出，展示当前结果和预期结果

## 模板中禁止出现的内容

- 可直接提交的完整答案
- 把核心算法完整写死在注释里
- 直接替用户完成思考过程的详细题解

## 验证要求

- 模板必须能够本地编译
- 即使核心函数还没写完，也应该能运行
- 运行输出要清楚表达“这是练习模板，还未完成真正测试”
