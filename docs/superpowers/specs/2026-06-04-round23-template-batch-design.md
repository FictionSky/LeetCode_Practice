# Round 2 / Round 3 模板批量生成设计

## 目标

把 Round 2 和 Round 3 的题目全部按统一规则准备到 `problems/` 目录下，并保持和当前练习模板一致：

- `md` 文件使用中文说明
- `cpp` 文件保留未作答模板
- 代码注释使用中文
- 本地运行输出使用中文

## 范围

本次包含：

- Round 2 全部题目
- Round 3 全部题目
- `002` 的标准化重建

本次不包含：

- 完整题解代码
- 自动抓取 LeetCode 全量题面
- 为每道题单独手写复杂测试数据

## 目录结构

每道题都统一为：

```text
problems/<题号三位补零>/<题号三位补零>.md
problems/<题号三位补零>/<题号三位补零>_<英文题名>.cpp
```

例如：

```text
problems/002/002.md
problems/002/002_AddTwoNumbers.cpp
problems/146/146_LRUCache.cpp
problems/297/297_SerializeAndDeserializeBinaryTree.cpp
```

## 模板规则

`md` 文件必须是中文练习说明，不直接给完整答案，只提供：

- 题目链接
- 中文题目要求区块
- 示例占位
- 约束占位
- 推荐题型
- 提示
- 复杂度目标

`cpp` 文件必须是练习版模板：

- 保留 `Solution` 或题目要求的设计类名称
- 保留本地 `main`
- 用中文注释明确告诉用户“哪里写代码”
- 不包含可直接提交的完整答案
- 运行输出使用中文，明确告诉用户当前还是练习模板

## 002 特殊要求

`002` 之前是旧结构中的已作答文件，不符合当前练习规则。本次要把它改成：

- `problems/002/002.md`
- `problems/002/002_AddTwoNumbers.cpp`

并删除旧的 `problems/002/002.cpp`。

## 生成策略

继续复用现有模板生成器，但把题库元数据拆出来，新增：

- Round 2 元数据
- Round 3 元数据
- 一个可按轮次生成的通用入口脚本

这样后续新增题目时，只需要补元数据即可继续生成。

## 验证

完成后至少验证：

- `002` 已标准化
- Round 2 / Round 3 代表题目录已生成
- `cmake -S . -B build` 可重新配置
- 至少各构建一个 Round 2 和 Round 3 代表目标
